HTML

Hypertext markup language(HTML) is the standard markup language for creating web pages or web applications.

Web browsers receive HTML documents from a web server or from local storage and render them into multimedia web pages.
HTML describes the structure of a web page semantically and originally included cues for the appearance of the document.
HTML elements are the building blocks of HTML pages. With HTML constructs, images and other objects, such as interactive forms, may be embedded into the rendered page.

it provides a means to create structured documents by denoting structural semantics for text such as headings, paragraphs, lists, links quotes and ohter items
HTML elements are delineated by tags, written using angle brackts.

위의 예문은 어떤 의미를 가지고 있는가 ?

1. 블로그, 카페, 커뮤니티 등에 편집기를 통해 글을 쓸 때, 모두가 HTML 코드를 생성하고 있었다는 것을 알 수 있다
2. 제목을 표현하기 위한 두 가지 수단은 아주 큰 차이점을 가지고 있다.

a. 구글 검색엔진을 통해 coding 이라는 검색한다.
b. 검색엔진은 전 세계 모든 웹페이지를 다운로드 받아 해당 웹사이트의 HTML코드를 분석하고,
c. 그 코드의 내용에서 이 웹페이지의 제목은 무엇이고 내용은 무엇인지를 코드를 통해 분류한다.

1번 예문이 사용된 정보는 검색엔진의 후순위에 배열되고, 2번 예문이 사용된 정보는 선순위에 배열될 것
어떤 두 사람이 각각 위의 방식으로 10년간 글을 써왔다고 가정한다면, 검색순위의 차이는 수십 수백배의 차이가 날 것이다.
'검색엔진에 노출되지 않는다' -> '실질적으로 존재하지 않는다' 라는 의미를 가짐.


웹이 가지고 있는 본래의 의미(=졍보)를 탄탄하게 만들기 위해서 HTML을 의미에 맞게 정확하게 사용해야 한다.
웹은 모든 운영체제에서 동작하고, 웹페이지 소스코드는 누구나 볼 수 있고, 웹은 저작권이 없는 순수한 공공재이다.
따라서 웹이 가지는 매우 중요한 의미 중 한 가지는 접근성 (accesibility)이다.
웹은 정보에 소외되는 사람이 없도록, 시각장애인의 경우 text를 인식하고 이를 청각 정보로 변환시켜주는 보조장치들(ex. screen reader)이 있다.
만약 니가 어떠한 정보를 보기 좋게 만들기 위해 image로 만든다면, 너의 정보는 접근성이 떨어지는 정보(시각장애인들에게 존재하지 않는)가 되는 것이다.
